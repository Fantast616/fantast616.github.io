一、什么是AOP？

​ 面向切面编程，利用 AOP 可以对业务逻辑的各个部分进行隔离，从而使得 业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率

​ 不通过修改源代码方式，在主干功能里面添加新功能

二、什么是代理模式？：

​ 代理模式的优势是可以很好地遵循设计模式中的开放封闭原则，对扩展开发，对修改关闭。你不需要关注目标类的实现细节，通过代理模式可以在不修改目标类的情况下，增强目标类功能的行为。

1、什么是静态代理？

​ 静态代理，就是通过声明一个明确的代理类来访问源对象。

​ 三个关键概念：

  • 抽象接口

  • 目标类

  • 代理类（必须包含真实对象类的对象）

2、什么是JDK动态代理？

​ 在程序执行过程中，使用jdk的反射机制，创建代理类对象，并动态的指定要代理的目标。

​ 动态代理是一种创建java对象的能力，让你不用创建TaoBao类，就能创建代理类对象。

​ 在java中，要想创建对象，正常流程如下：

  • 创建类文件，java文件编译为class文件
  • 使用构造方法，创建类的对象。

​ 在动态代理中，相当于把第一步的操作去掉了。通过反射包java.lang.reflect，里面有三个类：InvocationHandler，Method，Proxy。

jdk动态代理的实现：

  • InvocationHandler接口，有一个方法invoke()，代理类要完成的功能代码写在这个方法中即可。主要完成以下功能

    • 调用目标类的方法，完成目标类的功能

    • 功能增强，在目标方法调用时，增加一些其他的功能。

    • invoke方法的原型：

      • Object Proxy：jdk的代理对象，无需赋值，jdk会帮我们赋值
      • Method method：目标类中的方法，无需赋值，jdk会帮我们赋值
      • Object[] args：目标类中方法的参数

    • 如何使用？

      • 创建类实现接口InvocationHandler
      • 重写invoke()方法，把原来静态代理中代理类要完成的功能写在这部分即可。

  • Method类：表示方法的，确切的说就是目标类中的方法

    • 通过Method可以执行目标类的方法，通过method.invoke()
    • 等同于静态代理中的执行目标方法factory.sell(amount)

  • Proxy类：创建代理对象。使用Proxy类的方法，代替new的使用

    • 静态方法：newProxyInstance() ，用于代替原来的new UsbKingFactory()
    • public static Object newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)
      • ClassLoader：类加载器，负责向内存中加载对象，使用反射获取对象的ClassLoader。指目标对象的类加载器
      • Class<?>[] interfaces： 接口，目标对象实现的接口，也是反射获取的
      • InvocationHandler h：我们自己写的代理类要完成的功能类
      • 返回值就是代理对象

  • 具体实现：

    • 创建接口，定义目标类要完成的功能

    • 创建目标类实现接口

    • 创建InvocationHandler接口的实现类，在invoke方法中完成代理类的功能

      • 调用目标方法
      • 增强功能

    • 使用Proxy类的静态方法创建代理对象，并把返回值转为接口类型即可。

  • 如此一来，不同的代理类，比如淘宝、微商、京东等等都可以不用单独的创建类，直接利用JDK动态代理就可以了。

  • 故而，动态代理的作用：可以在不改变原来目标方法功能的前提下，可以在代理中增强自己的功能代码

  • 一些使用场景：比如：你所在的项目中，有一个功能是其他人（公司的其它部门，其它小组的人）写好的，你可以使用 GoNong.class，GoNong gn=new GoNong()，gn.print()；你发现这个功能，现在还存在一些缺点，不能完全满足项目的需要。我需要在gn.print()执行后，需要自己再增加代码。用代理实现gn.print()调用时，增加自己代理，而不用去改原来的GoNong文件。

3、什么是cglib动态代理？

​ JDK动态代理有一个前提，需要代理的类必须实现接口，如果潭有实现接口，只能通过CGLIB来实现，

​ cglib是第三方的工具库，创建代理对象。cglib的原理是继承，cglib通过继承目标类，创建它的子类，在子类中重写父类中同名的方法，实现功能的修改。所以其要求目标类不能是final的，方法也不能是final的。

  • 整体逻辑：

    • 通过继承的方式去获取到目标对象的方法
    • 通过传递方法拦截器MethodInterceptor实现方法拦截，在这里面做具体的增强
    • 调用生成的代理类对象具体执行重写的目标方法，直接调用方法拦截器里面的intercept方法
    • 前后加上了增强操作，从而实现了不修改目标类代码却实现业务增强的逻辑

三、AOP术语：

1、连接点

​ 类里面，哪些方法可以被增强，就称为连接点

2、切入点

​ 实际被增强的方法，称为切入点

3、通知（增强）

​ 实际增强的逻辑部分，称为通知。通知可以有如下类型：

  • 前置通知、后置通知
  • 环绕通知
  • 异常通知
  • 最终通知
4、切面

​ 这是一个动作，把通知应用到切入点这个过程，就叫做切面。
